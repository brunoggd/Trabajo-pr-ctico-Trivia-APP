Trabajo Práctico:
Trivia App
Aplicación de preguntas y respuestas usando clases JavaScript, fetch y manejo de errores con la Open Trivia API.
Modalidad: Individual o parejas
Entrega: 2 semanas
Archivos: JS + HTML separados
Puntaje: 100 pts + 20 bonus
Sección 01
Contexto y objetivo

Van a construir una aplicación de trivia que consume la Open Trivia Database, una API pública y gratuita que devuelve preguntas de múltiple opción sobre distintas categorías.

El objetivo del TP no es solo que funcione, sino que el código esté organizado en clases, que los errores estén correctamente manejados y que la interfaz sea clara para el usuario.

API base
https://opentdb.com/api.php?amount=10&type=multiple
📖 Cómo funciona la API
La API devuelve un objeto con un array results. Cada elemento tiene: question, correct_answer, incorrect_answers (array de 3) y category. Las preguntas tienen caracteres HTML escapados (&amp;, &quot;) — hay que decodificarlos.
Respuesta de ejemplo — opentdb.com
JSON
{
  "response_code": 0,
  "results": [
    {
      "category": "Science & Nature",
      "difficulty": "medium",
      "question": "What is the chemical symbol for gold?",
      "correct_answer": "Au",
      "incorrect_answers": ["Ag", "Fe", "Cu"]
    }
  ]
}
Sección 02
Estructura de archivos

El proyecto se entrega con esta estructura de carpetas y archivos:

trivia-app/
  ├── index.html  // estructura y UI
  ├── css/
  │   └── styles.css  // estilos
  └── js/
      ├── TriviaAPI.js  // clase que maneja fetch
      ├── TriviaGame.js  // clase que maneja la lógica del juego
      └── main.js  // punto de entrada, conecta todo
⚠ Obligatorio
La lógica de fetch va en TriviaAPI.js y la lógica del juego en TriviaGame.js. No mezclar todo en un solo archivo. Esa separación es parte de la nota.
Sección 03
Consignas
  • 1
    Clase TriviaAPI
    Crear la clase TriviaAPI en js/TriviaAPI.js con al menos los siguientes métodos:
    • getPreguntas(cantidad, categoria, dificultad) — hace el fetch y devuelve el array de preguntas
    • getCategorias() — trae las categorías disponibles desde https://opentdb.com/api_category.php
    Debe incluir manejo de errores con try/catch y verificar response.ok.
    OBLIGATORIO
  • 2
    Clase TriviaGame
    Crear la clase TriviaGame en js/TriviaGame.js con:
    • Propiedad puntaje (inicia en 0)
    • Propiedad preguntaActual (índice)
    • Método iniciar(preguntas) — carga las preguntas y resetea el estado
    • Método responder(respuesta) — verifica si es correcta, actualiza puntaje y devuelve true/false
    • Método siguiente() — avanza a la próxima pregunta
    • Método haTerminado() — devuelve true si se respondieron todas
    OBLIGATORIO
  • 3
    Decodificación de HTML
    Las preguntas de la API vienen con caracteres escapados. Implementar una función utilitaria decodificarHTML(texto) que los convierta correctamente antes de mostrarlos.

    Pista: se puede usar un elemento <textarea> temporal del DOM para decodificar.
    OBLIGATORIO
  • 4
    Interfaz de usuario
    El index.html debe mostrar:
    • Pantalla de inicio con selector de dificultad (fácil / medio / difícil) y botón "Jugar"
    • Texto de la pregunta actual y número de pregunta (ej: "Pregunta 3 de 10")
    • 4 opciones de respuesta como botones (las respuestas deben aparecer en orden aleatorio)
    • Feedback visual al responder: verde si acertó, rojo si falló
    • Puntaje visible en todo momento
    • Pantalla final con puntaje total y botón para volver a jugar
    OBLIGATORIO
  • 5
    Estados de carga y error
    Mostrar al usuario:
    • Un mensaje de "Cargando preguntas..." mientras se hace el fetch
    • Un mensaje de error claro si la API falla (sin mostrar el error técnico crudo)
    • Un botón "Reintentar" cuando hay error
    OBLIGATORIO
  • 6
    Selector de categoría
    Usar getCategorias() para popular dinámicamente un <select> con las categorías disponibles. El jugador puede elegir una antes de empezar.
    BONUS +10
  • 7
    Historial de partidas
    Guardar en localStorage los últimos 5 puntajes con fecha. Mostrarlos en la pantalla final.
    BONUS +10
